# Skill：API 契约（v0.2）

## 必须内容
- endpoint + method + auth（如有）
- request/response JSON 示例（字段类型与可选性）
- error codes（统一风格与含义）
- 兼容策略（新增字段默认可选，旧字段不删）

## 规则
- Response：新增字段必须可选；客户端必须容忍未知字段
- Request：服务端必须容忍旧字段缺失；必要时给默认值
- 错误码：新增可以；修改含义不允许；code 含义稳定

## MR 必须附"契约差异摘要"
- 删除字段：无/有（列出）
- 重命名字段：无/有（列出，默认不允许）
- 新增字段：列出 + 可选性 + 默认值
- 枚举/状态变化：列出 + Unknown/Default 兜底
