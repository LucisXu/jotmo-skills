# Jotmo Skills - AI 辅助开发知识库

## 这是什么

这是 Jotmo 团队的 **AI 辅助开发知识库**。通过这个知识库，团队中的任何人都可以借助 Claude 完成开发工作。

**核心理念**：把"需求 → 实现 → 测试 → 上线"变成一条人人都能走的标准路径。

---

## 谁可以用

| 角色 | 能做什么 |
|------|----------|
| 产品经理 | 将需求转化为清晰的规格文档，与 Claude 对话完善需求细节 |
| 设计师 | 描述交互逻辑，让 Claude 协助生成技术方案 |
| 前端开发 | 实现页面和交互，Claude 会遵循团队规范写代码 |
| 后端开发 | 实现接口和服务，Claude 会遵循架构约定写代码 |
| 全栈/独立开发 | 端到端完成功能，Claude 协调前后端配合 |

---

## 快速开始（3 步）

### 第 1 步：明确你要做什么

在和 Claude 对话前，先想清楚：
- 我要解决什么问题？
- 用户会怎么用这个功能？
- 有什么必须满足的约束？

### 第 2 步：选择合适的起手式

打开 `CLAUDE_PROMPTS.md`，找到适合你场景的提示词模板，复制后开始对话。

| 我是谁 | 我要做什么 | 用哪个模板 |
|--------|-----------|-----------|
| 产品/设计 | 把想法变成规格文档 | **规格访谈** |
| 任何人 | 技术选型或不确定怎么做 | **技术决策** |
| 前端开发 | 实现页面或组件 | **前端开发** |
| 后端开发 | 实现接口或服务 | **后端开发** |
| 开发者 | 提交代码前检查 | **MR 准备** |

### 第 3 步：对话中确认再动手

Claude 会先输出计划，**等你确认后才写代码**。这样可以：
- 确保方向正确，避免返工
- 保证符合团队规范
- 让非技术同学也能参与把关

## 常见场景速查

### 场景 1：我有个需求，不知道怎么开始

1. 打开 `CLAUDE_PROMPTS.md`，使用"规格访谈"模板
2. 把你的想法告诉 Claude
3. Claude 会追问细节，帮你整理出规格文档
4. 确认规格后，Claude 会推荐下一步该怎么做

### 场景 2：我要开发一个新功能

1. 根据你的角色选择模板（前端/后端/全栈）
2. 告诉 Claude 你要做什么
3. Claude 会先出计划，你确认后再写代码
4. 代码完成后，Claude 会帮你准备 MR

### 场景 3：我不确定技术方案怎么选

1. 使用"技术决策"模板
2. 描述你的问题和约束
3. Claude 会对比分析选项，给出推荐
4. 如果偏离默认方案，需要写 ADR 文档

### 场景 4：我要提交代码

1. 使用"MR 准备"模板
2. Claude 会按规范生成 MR 描述
3. 确保包含所有必填项后再提交

## 团队底线（必须遵守）

无论谁做开发，都要确保：

1. **线上稳定** - 不引入已知问题
2. **向后兼容** - 老用户、老数据不受影响
3. **可灰度** - 能小范围先试
4. **可回滚** - 出问题能快速恢复
5. **可观测** - 知道系统在发生什么

这些规则写在 `CLAUDE.md` 中，Claude 会自动遵守。

## 遇到问题怎么办

### 问：Claude 写的代码不对怎么办？

告诉 Claude 哪里不对，它会修正。如果反复不对，检查：
- 需求描述是否清晰
- 是否遗漏了重要约束

### 问：Claude 的建议和我想的不一样

Claude 会按团队规范执行。如果你认为规范有问题：
1. 先按规范完成任务
2. 事后提出规范改进建议

### 问：我的场景没有对应的模板

使用"规格访谈"模板，描述你的需求，Claude 会帮你找到合适的方式。

## 持续改进

每次遇到问题，问三个问题：

1. **是规则缺失？** → 补充到 `CLAUDE.md`
2. **是套路缺失？** → 补充到 `skills/`
3. **是文档过时？** → 更新 `docs/`

这样知识库会越用越完善。
